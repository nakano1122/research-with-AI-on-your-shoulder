# Idea Organizer

ユーザーのアイデアを整理・構造化し、図解で可視化する。

## ワークフロー

```
1. メモ収集
   → content/daily/ から関連メモを検索
   → ユーザーの現在の考えを聞く
   
2. 構造化
   → アイデアを「問い→仮説→検証方法」で整理
   → 関連アイデア間のリンクを発見
   
3. 可視化
   → /drawio でマインドマップや概念図を作成
   → generated/ ディレクトリに保存
   
4. フィードバック
   → 整理結果をユーザーに確認
   → 不足点や新しい気づきを議論
```

## メモ参照

```bash
# 最近のメモを確認
Glob content/daily/*/memo.md
Grep "アイデア|仮説|課題" content/daily/
```

## 構造化フレームワーク

アイデアを以下の観点で整理:

| 観点 | 質問 |
|------|------|
| What | 何を解決したいか？ |
| Why | なぜそれが重要か？ |
| How | どうやって解決するか？ |
| Diff | 既存手法との違いは？ |

## 図解作成

`/drawio` スキルを使用して図を作成:

```
/drawio マインドマップを作成
- 中心: メインアイデア
- 分岐: サブアイデア、関連概念
- 色分け: カテゴリ別
```

出力先: `content/daily/{現在のメモ}/generated/idea-map.drawio`

## 出力形式

整理結果をMarkdown + 図解で出力:

```markdown
# アイデア整理: {タイトル}

## 核心的な問い
{1文で表現}

## 仮説
1. ...
2. ...

## 検証方法
- ...

## 関連するアイデア
- [過去のメモへのリンク]

## 図解
![アイデアマップ](generated/idea-map.drawio)
```

## 対話のコツ

- 「それはどういう意味ですか？」と深掘り
- 「他に関連することはありますか？」と広げる
- 「一番大事なことは何ですか？」と絞り込む
