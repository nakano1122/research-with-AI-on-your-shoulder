# AutoEncoder系の次元圧縮手法

生成的推薦や検索でアイテム/文書をトークン列に変換する際の基盤技術。

## AutoEncoder (AE)

入力ベクトルを低次元に圧縮し、再び高次元に復元することで学習する次元圧縮モデル。

- **Encoder**: 高次元 → 低次元（圧縮）
- **Decoder**: 低次元 → 高次元（復元）
- **学習**: 再構成誤差を最小化

**弱点**: 外れ値的なベクトルの場合、圧縮時に重要な情報が均されてしまう。

## Variational AutoEncoder (VAE)

AEの弱点に対応。圧縮後の表現を確率分布として扱う。

- 入力ベクトルの各次元について「平均」「標準偏差」を出力
- 乱数をサンプリングして圧縮ベクトルを生成
- **学習**: 再構成誤差 + KLダイバージェンス（圧縮ベクトルが標準正規分布に従うよう正則化）

**弱点**: 圧縮後ベクトルが粗い粒度でしか情報を表現できない。

## Residual-Quantized VAE (RQ-VAE)

VAEの粗い粒度問題に対応。残差を繰り返し量子化することで細かい表現を可能にする。

1. グローバルなcodebook（クラスタ中心の集合）を用意
2. 入力ベクトルに最も近いクラスタ中心を選択 → 1つ目のトークン
3. 入力とクラスタ中心の**残差**を次のステップに渡す
4. 残差に対して同じcodebookで量子化を繰り返す → 2つ目、3つ目...のトークン

**弱点**: グローバルなcodebookを使うため、似たアイテム同士の微細な違いを表現しにくい。トークン分布が偏りやすい。

## Balanced Semantic ID (MBGen)

RQ-VAEの弱点に対応。各ステップで独立したクラスタリングを行う。

- 1つ目: 全体をK個にクラスタリング
- 2つ目: 各クラスタ内で独立してk-means → 分布が均衡化
- 3つ目: 衝突回避のためランダム割り当て

グローバルcodebookではなく、階層ごとに独立したクラスタリングを行うことで、似たアイテム同士の微細な違いを表現可能に。

## 比較

| 手法         | 出力                     | 特徴                       |
| ------------ | ------------------------ | -------------------------- |
| AE           | 連続ベクトル             | シンプル、外れ値に弱い     |
| VAE          | 連続ベクトル（確率分布） | 正則化あり、粗い粒度       |
| RQ-VAE       | 離散トークン列           | 細かい表現可能、分布が偏る |
| Balanced SID | 離散トークン列           | 分布均衡、トークン数削減   |
